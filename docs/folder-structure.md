# Folder Structure

This is the structure of your generated project (after scaffolding or using the template).

```
your-app-tests/
│
├── tests/
│   ├── baseline/          ← page load + element visibility tests
│   │   ├── guest/         ← tests for unauthenticated users
│   │   ├── user/          ← tests for logged-in users
│   │   └── admin/         ← tests for admin users
│   ├── e2e/               ← full user flow tests
│   ├── regression/        ← edge cases and negative tests
│   └── marketing/         ← landing page and public page tests
│
├── page-objects/          ← Page Object Models (one class per page)
│   ├── base/
│   │   └── base-page.ts   ← base class all Page Objects extend
│   ├── guest/             ← page objects for guest-visible pages
│   ├── user/              ← page objects for user-visible pages
│   └── admin/             ← page objects for admin pages
│
├── helpers/               ← shared utilities used across tests
│   ├── navigation.ts      ← gotoWithRetry() — use instead of page.goto()
│   ├── cloudflare.ts      ← handles Cloudflare challenge pages
│   ├── rate-limiter.ts    ← prevents hammering the app during scans
│   └── env.ts             ← type-safe .env accessors
│
├── fixtures/
│   └── index.ts           ← custom Playwright fixtures (userPage, adminPage)
│
├── constants/
│   └── routes.ts          ← all app URLs in one place — never hardcode URLs
│
├── types/
│   └── index.ts           ← shared TypeScript types
│
├── utils/
│   └── date.ts            ← date helpers for test data generation
│
├── setup/
│   └── auth.setup.ts      ← auth session creator — run once before tests
│
├── prompts/               ← copy-paste prompts for your AI agent
│   ├── 00-quick-start.md  ← start here — the full pipeline in one prompt
│   ├── 01-scan-pages.md
│   ├── 02-scan-codebase.md
│   ├── 03-generate-baseline.md
│   ├── 04-generate-e2e.md
│   ├── 05-generate-regression.md
│   ├── 06-fix-errors.md
│   ├── 07-cleanup.md
│   └── 08-rescan.md
│
├── docs/                  ← documentation (you are here)
│
├── page-maps/             ← GENERATED — gitignored
│   └── <role>/<page>.json ← page maps saved by the scan prompt
│
├── code-context/          ← GENERATED — gitignored
│   ├── summary.json
│   ├── routes.json
│   └── components.json
│
├── auth/                  ← GENERATED — gitignored — NEVER COMMIT
│   ├── user-state.json    ← saved auth session for regular user
│   └── admin-state.json   ← saved auth session for admin
│
├── snapshots/             ← GENERATED — gitignored
│
├── test-results/          ← GENERATED — gitignored
│
├── playwright-report/     ← GENERATED — gitignored
│
├── playwright.config.ts   ← Playwright configuration
├── tsconfig.json          ← TypeScript configuration
├── package.json
└── .env                   ← your config — gitignored
```

## Which files to edit

| Task | File to edit |
|------|-------------|
| Add a new page to test | Create `page-objects/<role>/<page>.ts`, then `tests/baseline/<role>/<page>.spec.ts` |
| Add a new route | `constants/routes.ts` |
| Change Playwright settings | `playwright.config.ts` |
| Add a shared utility | `helpers/` |
| Update auth logic | `setup/auth.setup.ts` |

## Which files NOT to edit manually

- `page-maps/` — regenerated by scan prompt
- `code-context/` — regenerated by codebase scan prompt
- `auth/` — regenerated by `npm run auth`
- `playwright-report/` — regenerated after each test run
